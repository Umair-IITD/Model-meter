# PRD.md — Model-meter: AI Spend Audit Tool

> Source of truth: Credex Assignment PDF (12 pages). Research brief used as supporting context only. Where they conflict, the assignment wins.

---

## 1. Product Overview

Model-meter is a free web application that audits what a startup or team currently spends on AI tools, identifies overspend against official vendor pricing, and surfaces actionable recommendations with quantified savings. It functions as a lead-generation asset for Credex (which resells discounted AI credits) while delivering genuine standalone value to any user.

**Positioning in one sentence:** "Mint for AI tool spend" — a FinOps instrument that a finance-literate person can trust.

---

## 2. Target User

**Primary:** Engineering Manager or Technical Co-founder at a Series A/B startup (20–100 engineers). They are seeing AI tooling costs exceed AWS spend for the first time and are under pressure from finance to justify it. They are afraid that cutting AI budgets will hurt developer productivity. They need an authoritative, per-seat breakdown they can show a CFO.

**Secondary:** Seed/Pre-Seed founder paying for multiple individual subscriptions (Cursor + ChatGPT + Claude) on personal credit cards. The "Shadow AI consolidation" narrative is their hook.

**Who is NOT the target:** Enterprise procurement teams, individual consumers with no team context, non-technical users who can't answer "which plan are you on."

---

## 3. Core User Journey

```
Landing page
    ↓
Spend input form (no login required)
    ↓
Instant on-screen audit results + AI-generated summary
    ↓
Email capture (shown AFTER value is delivered, never before)
    ↓  [if >$500/mo savings]
Credex consultation CTA
    ↓  [if <$100/mo savings or already optimal]
"Notify me" signup with honest messaging
    ↓
Shareable unique public URL (with OG preview, PII stripped)
```

**Critical constraint:** Email is collected after the audit is shown. No gate before value. This is stated explicitly in the assignment and is also the correct product decision.

## 4. MVP Scope — All 6 Features Required

### Feature 1: Spend Input Form

**Required tools (minimum — all must be present at submission):**

| Tool | Plans to Support |
|------|-----------------|
| Cursor | Hobby / Pro / Pro+ / Ultra / Business |
| GitHub Copilot | Free / Pro / Pro+ / Business / Enterprise |
| Claude (Web) | Free / Pro / Max / Team / Enterprise / API direct |
| ChatGPT | Free / Go / Plus / Pro / Business / Enterprise / API direct |
| Anthropic API direct | (token-based, not seat-based) |
| OpenAI API direct | (token-based) |
| Gemini | Pro / Ultra / API |
| Windsurf OR v0 | Pick one — include both if time allows, pick Windsurf first (more developer-relevant) |

**Per-tool inputs:**
- Which plan
- Current monthly spend (user-entered USD amount — do not rely solely on calculated pricing)
- Number of seats

**Global inputs:**
- Team size
- Primary use case: `coding` / `writing` / `data` / `research` / `mixed`

**State persistence:** Form state must survive page reload. Implementation: `localStorage` keyed to a session ID. Do NOT use URL params for this (too noisy for complex multi-tool state). See DESIGN_DOC for detail.

### Feature 2: Audit Engine

The engine evaluates each tool the user has entered and produces per-tool findings. It must pass the "finance person reads the reasoning and agrees" test.

**Rule types (see AUDIT_ENGINE_SPEC.md for full logic):**
- Plan-fit rules (right plan for team size?)
- Same-vendor downgrade rules (cheaper plan, same capability?)
- Cross-tool substitution rules (cheaper tool, same use-case fit?)
- Credex arbitrage signal (paying retail when credits exist?)

**Critical constraint from assignment:** "For the audit math itself, hardcoded rules are correct — knowing when not to use AI is part of the test." The engine must be deterministic rule-based logic. No LLM in the audit calculation path.

**Pricing data requirement:** Every number must trace to an official vendor URL, cited in `PRICING_DATA.md`. Do not use research brief pricing without verifying against official pages.

### Feature 3: Audit Results Page

- Per-tool breakdown: current spend → recommended action → projected savings → one-sentence reason
- Hero section: total monthly savings + total annual savings, large and prominent
- For audits showing **>$500/mo savings**: surface Credex CTA prominently
- For audits showing **<$100/mo savings or already-optimal**: be honest — "You're spending well." Still capture lead with "notify me" framing
- Visual quality matters: this page is the thing that gets screenshotted and shared

### Feature 4: AI-Generated Personalized Summary

- ~100-word paragraph generated by Anthropic API (preferred) or any LLM
- Called after audit engine runs, not before
- Must handle API failures gracefully with a templated fallback
- Full prompt must be in `PROMPTS.md`
- This is the ONLY feature where LLM is used in the data path

### Feature 5: Lead Capture + Storage

- Email (required) + optional fields: company name, role, team size
- Stored in Firebase Firestore (see TECH_STACK.md for rationale)
- Transactional email sent on capture: Resend (preferred) or Postmark
- Email confirms the audit and flags Credex outreach for high-savings cases
- Abuse protection: honeypot field + rate limiting on the API route. Document choice in DESIGN_DOC.

### Feature 6: Shareable Result URL

- Each completed audit gets a unique UUID-based public URL: `/audit/[uuid]`
- Public version strips: email, company name, role
- Public version shows: tools, plans, savings numbers, recommendations
- Open Graph tags + Twitter Card meta tags on each audit page
- Dynamic OG image generated via `next/og` showing the total savings number

## 5. Non-Goals (MVP)

- User authentication or accounts
- PDF export (bonus only)
- Embeddable widget (bonus only)
- Benchmark mode comparing to industry averages (bonus only)
- Referral codes (bonus only)

## 7. Key Edge Cases

| Scenario | Required Behavior |
|----------|-------------------|
| User enters $0 spend for a paid plan | Flag as inconsistency, show neutral finding |
| User enters only one tool | Audit still works, just for that tool |
| User selects API direct (no seats) | Show token-rate analysis, ask for estimated monthly spend |
| All tools already optimal | Show "You're spending well" — do not manufacture savings |
| Savings between $100–$500 | Standard recommendations, no special Credex CTA |
| Anthropic API failure during summary | Fall back to template summary silently, never show an error on the result page |
| UUID not found (bad link) | 404 page with link back to the form |
| No email submitted (user skips capture) | Audit still accessible, just no lead stored |
